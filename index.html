What this parser expects

  • It scans every sheet, not just the first one.
  • A row becomes a header candidate if it matches at least 3 target fields.
  • Column names are matched case-insensitively.
  • Spaces, underscores, and punctuation differences are ignored.
  • Rows with all six fields empty are skipped.

How the workflow moves

This is the visual explanation section you asked for. Three stages. No mystical AI smoke machine nonsense.

01

Scan workbook

The browser reads the uploaded workbook and inspects every sheet independently. Blank rows above the table are fine.

02

Match headers

Candidate header rows are scored based on how many normalized target fields they match. The best row on each sheet wins.

03

Surface clean records

Rows below the chosen header are extracted into structured issue records, then exposed in a searchable dashboard with details and export.

Publish online

GitHub Pages setup guide

Since you want this entirely online, this section exists so future-you doesn’t forget the clicks and then act betrayed by a settings menu.

Easiest upload method

  1. Create a new public GitHub repository.
  2. Add index.html, styles.css, script.js, and README.md.
  3. Open Settings in the repo.
  4. Go to Pages.
  5. Under Build and deployment, choose Deploy from a branch.
  6. Select the main branch and the root folder.
  7. Save, then wait for GitHub to publish the site.

What the live URL looks like

Your site will usually be published at:

https://YOUR-USERNAME.github.io/REPOSITORY-NAME/

Example:

https://johndoe.github.io/issue-explorer/
